Graphical interfaces

The DataPower® Gateway provides two graphical interfaces: WebGUI and Blueprint Console.

You can use both graphical interfaces to manage the DataPower Gateway, develop and troubleshoot services, and monitor status. However, the following tasks are available from only the Blueprint Console:
  • Creating and managing service patterns
  • Creating tenants.

Independent of the graphical interface, both manage the configuration of objects. An object can reference another object that can, in turn, reference another object. For example, a service configuration references an XML manager that references a user agent. To meet your business-processing criteria and security requirements, the DataPower Gateway provides flexibility to manage the base configurations and their referenced configurations.

Objects that you can configure on the DataPower Gateway range from simple to complex. An object is any entity that you configure on the DataPower Gateway. All objects provide an object view, or a generic configuration. However, many DataPower services provide an object view and a service view. For a Multi-Protocol Gateway, Web Service Proxy, and Access Manager Reverse Proxy, you can use patterns.

  • The object view allows expert-level developers to build specific, complex, and detailed objects.
  • The service view allows less-than-expert level developers to build basic objects. The service view is a wizard that steps the developer through the creation of an object. The service view might not include all configuration properties. The service view is available in only the WebGUI.
  • Patterns allow developers to create services from base service configurations. Patterns do not include all configuration properties to the pattern developer or the pattern deployer.
