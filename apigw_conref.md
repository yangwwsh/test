 ## Referencing common steps {#commonsteps}

1.   In the search field, enter API. 
2.   In the search field, enter assembly. 
3.   Click **Add** or **New**. 
4.   Define the basic properties: Name, administrative state, and descriptive summary. 
5.   In the **Name** field, enter the name for the configuration. 
6.   Set the administrative state of the configuration. 
7.   In the **Comments** field, enter a descriptive summary. 
8.   Click **Apply** to save the changes to the running configuration. 
9.   Click **Save Configuration** or **Save changes** to save the changes to the persisted configuration. 
10.  Use in substeps. 
    1.   Click **Add** or **New**. 
    2.   Click **Apply**. 
    3.   Define the basic properties: Name, administrative state, and descriptive summary. 
    4.   In the **Name** field, enter the name for the configuration. 
    5.   Set the administrative state of the configuration. 
    6.   In the **Comments** field, enter a descriptive summary. 
    7.   Click **Apply** to save the changes to the running configuration. 
    8.   Click **Save Configuration** or **Save changes** to save the changes to the persisted configuration. 
11.  Use in ordered lists. 
    1.  Click **Add** or **New**.
    2.  Click **Apply**.
    3.  In the **Name** field, enter the name for the configuration.
    4.  Set the administrative state of the configuration.
    5.  In the **Comments** field, enter a descriptive summary.
    6.  Click **Apply** to save the changes to the running configuration.
    7.  Click **Save Configuration** or **Save changes** to save the changes to the persisted configuration.

## Referencing action steps {#actionconfiguration}

1.   Drag the **Advanced** icon to the configuration path. 
2.   Double-click the **Advanced** icon. 
3.   Click the **Advanced** tab. 
4.   To enable debug for the action, click **Enable GatewayScript Debug**. 
5.   Click **Enable GatewayScript Debug** to debug a GatewayScript program. The program can be called from a `gatewayscript` extension element or a gatewayscript\(\) extension function in the stylesheet that you specify in the **Transform File** field.

If this action is the last one for the rule, click **Apply Policy**. Otherwise, drag another icon to the configuration path.
